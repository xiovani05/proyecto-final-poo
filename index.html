1. Introducción al paradigma de la POO

1.1 Elementos del modelo de objetos

Clases: Moldes o plantillas abstractas que definen atributos y métodos.
Objetos: Instancias concretas de una clase con estado y comportamiento propio.
Abstracción: Aislamiento de características esenciales omitiendo detalles complejos.
Modularidad: Subdivisión del software en partes independientes y manejables.
Encapsulamiento: Ocultamiento de datos internos accesibles solo mediante interfaces públicas.
Herencia: Mecanismo por el cual una clase hija adquiere propiedades de una clase padre.
Polimorfismo: Capacidad de responder al mismo llamado de formas distintas según la naturaleza del objeto.

1.2 Lenguaje de modelado unificado: diagrama de clases

UML es un estándar visual para diseñar sistemas. Una clase se divide en: Superior (Nombre), Central (Atributos) e Inferior (Métodos).

Visibilidad: + Público, - Privado, # Protegido.

2. Clases y Objetos

2.1 Declaración de clases: atributos, métodos, encapsulamiento

Atributos: Variables que definen las características del objeto.
Métodos: Funciones que representan las acciones del objeto.
Encapsulamiento: Restricción de acceso directo a los atributos usando modificadores.

2.2 Instanciación de una clase

Proceso de crear un objeto en memoria a partir de una clase mediante la palabra clave new.

2.3 Referencia al objeto actual

Permite a los métodos referirse al objeto mismo usando palabras como this o self.

2.4 Métodos: declaración, mensajes, paso de parámetros, retorno

Involucra la definición del método, la llamada (mensaje), los argumentos enviados y el resultado devuelto (return).

2.5 Constructores y destructores: declaración, uso y aplicaciones

Los constructores inicializan el objeto al crearse. Los destructores liberan los recursos asignados en memoria.

2.6 Sobrecarga de métodos

Definir varios métodos con el mismo nombre pero con diferentes parámetros.

2.7 Sobrecarga de operadores

Redefinir el comportamiento de operadores estándar (+, -, ==) para objetos personalizados, dividiéndose en unarios y binarios.

3. Herencia

3.1 Definición: clase base, clase derivada

Clase Base: Clase genérica original con atributos y métodos comunes.
Clase Derivada: Nueva clase que hereda el comportamiento y añade rasgos específicos.

3.2 Classification: herencia simple, herencia múltiple

La herencia simple se da cuando se hereda de un solo padre. La múltiple hereda de dos o más clases bases simultáneamente.

3.3 Reutilización de miembros heredados

Las clases hijas acceden automáticamente al código de la clase padre sin necesidad de reescribirlo.

3.4 Referencia al objeto de la clase base

Uso de palabras clave como super o base para invocar constructores o métodos heredados.

3.5 Constructores y destructores en clases derivadas

Primero se ejecuta el constructor de la clase base y luego el de la clase derivada. Al destruir el objeto, el orden es inverso.

3.6 Redefinición de métodos en clases derivadas (Overriding)

Capacidad de una clase hija de reescribir e implementar un método que ya existe en la clase padre.

4. Polimorfismo

4.1 Definición

Permite tratar a múltiples objetos de clases derivadas como si fueran de la clase base común, ejecutando la acción que corresponda a cada uno.

4.2 Clases abstractas: definición, métodos abstractos, implementación y modelado

Clases que no se pueden instanciar y sirven de guía. Los métodos abstractos no tienen código y obligan a las clases hijas a implementarlos.

4.3 Interfaces: definición, implementación de interfaces, herencia de interfaces

Contratos abstractos que definen qué debe hacer una clase sin explicar el cómo, permitiendo simular herencia múltiple.

4.4 Variables polimórficas (plantillas): definición, uso y aplicaciones

Variables de tipo clase base que pueden almacenar objetos de cualquiera de sus clases hijas en tiempo de ejecución.

4.5 Reutilización de código

Permite diseñar sistemas genéricos y scalables que acepten nuevas clases futuras sin modificar el código actual.

5. Excepciones

5.1 Definición

Eventos u objetos que se generan cuando ocurre un error durante la ejecución, interrumpiendo el flujo normal del sistema.

5.2 Tipos de excepciones

Verificadas (Checked): el compilador obliga a manejarlas antes de ejecutar. No verificadas (Runtime): errores lógicos en ejecución.

5.3 Propagación de excepciones

Si un error no se controla en un método, viaja hacia arriba en la pila de llamadas buscando un bloque que lo resuelva.

5.4 Gestión de excepciones: manejo, lanzamiento

Controlado mediante los bloques estructurados: try (prueba), catch (captura) y throw (lanzamiento manual).

6. Flujos y Archivos

Mecanismos que permiten a una aplicación interactuar con fuentes de almacenamiento externas, logrando que los datos sobrevivan a la ejecución del programa mediante la persistencia en disco duro u otros medios.

6.1 Definición

Un Flujo (Stream) es una abstracción conceptual que representa una secuencia ordenada de datos con un origen (fuente de lectura) y un destino (sumidero de escritura). Permite transferir información de manera continua, byte por byte o carácter por carácter, sin necesidad de cargar todo el archivo en la memoria RAM al mismo tiempo.

6.2 Clasificación: Archivos de texto y binarios

Archivos de Texto: Almacenan datos estructurados como secuencias de caracteres legibles por humanos (ej. extensiones .txt, .csv, .json). Se codifican utilizando estándares de caracteres como UTF-8 o ASCII.
Archivos Binarios: Almacenan información en formato binario puro (secuencias de ceros y unos) optimizada para la computadora (ej. imágenes .png, ejecutables .exe, o bases de datos compactas). No son legibles de forma directa mediante un editor de texto convencional.

6.3 Operaciones básicas y tipos de acceso

Cualquier interacción con el sistema de archivos sigue un ciclo de vida estrictamente regulado por el sistema operativo:

  • Operaciones Básicas:
    • Abrir (Open): Establece un canal de comunicación seguro y verifica los permisos del archivo.
    • Leer (Read): Extrae información desde el soporte físico hacia variables dentro del programa.
    • Escribir (Write): Guarda datos temporales de la aplicación en el almacenamiento permanente.
    • Cerrar (Close): Libera los recursos del sistema operativo asignados al archivo (vital para evitar archivos corruptos o bloqueados).
  • Tipos de Acceso:
    • Acceso Secuencial: Los datos se leen o escriben en orden estricto desde el principio hasta el final del archivo. Para llegar al dato 'N', hay que pasar obligatoriamente por los 'N-1' anteriores.
    • Acceso Aleatorio (Directo): Permite posicionar el puntero de lectura/escritura en cualquier byte del archivo instantáneamente mediante un índice o posición numérica, ideal para sistemas de archivos de bases de datos.

6.4 Manejo de objetos persistentes

Es la técnica que permite guardar el estado actual de un objeto completo (atributos con sus respectivos valores en un momento dado) directamente en un medio de almacenamiento físico, para poder restaurarlo idénticamente en el futuro.

  • Serialización: El proceso de transformar un objeto de la memoria RAM en una secuencia lineal de bytes lista para ser enviada por red o grabada en un archivo físico.
  • Deserialización: El proceso inverso, donde se toma una secuencia de bytes almacenada en disco y se reconstruye fielmente el objeto original con toda su estructura lógica en la memoria de la aplicación.
